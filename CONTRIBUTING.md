# المساهمة في روش ✦

أهلًا بيك! أي مساهمة — حتى لو تصليح كلمة غلط — بتفرق. الملف ده بيوضحلك إزاي تشتغل على المشروع من غير لخبطة.

---

## ✦ ابدأ في دقيقتين

```bash
git clone https://github.com/Lord-shaban/rawsh.git
cd rawsh
npm install      # بس عشان أدوات الفحص — المشروع نفسه مالوش build
npm run dev      # http://localhost:8080
```

> **مهم:** المشروع **مالوش build step**. الملفات اللي في `js/` و `css/` هي بالظبط اللي بيشتغل في المتصفح. عدّل واعمل refresh وخلاص.

بس لازم سيرفر محلي (مش هيشتغل بفتح `index.html` مباشرة) لأن المشروع بيستخدم ES Modules.

---

## ✦ فلسفة المشروع

قبل ما تكتب سطر، افهم القواعد الأربعة دي:

| القاعدة | ليه |
| --- | --- |
| **صفر build، صفر framework** | حد يفتح الريبو ويفهمه في ساعة، ويرفعه على أي استضافة static في دقيقة |
| **المنطق في قاعدة البيانات** | العدادات والإشعارات والهاشتاجات كلها Triggers في Postgres — الكلاينت مايحسبش ومايزوّدش عدادات بنفسه |
| **الأمان في RLS** | أي جدول جديد لازم يتفعّل عليه RLS بسياسات واضحة. الـ anon key عام بطبيعته — الحماية من الداتابيز |
| **الشكل جزء من المنتج** | ده مش مجرد CRUD. أي ميزة جديدة لازم تحس إنها من نفس عالم البوسترات |

## ✦ ازاي تشتغل

### 1. افتح issue الأول
لو حاجة كبيرة، اتكلم قبل ما تكتب. لو تصليح صغير، ادخل على طول.

### 2. اعمل branch
```bash
git checkout -b feat/story-replies
```

الأسماء: `feat/…` · `fix/…` · `docs/…` · `style/…` · `refactor/…` · `db/…`

### 3. اكتب الكود

**ستايل الكود:**
- مسافتين للمسافة البادئة، فاصلة منقوطة في آخر السطر، علامات تنصيص مفردة `'`
- **التعليقات بالعربي** — ده مشروع عربي، والتعليقات للناس اللي هتقراه
- أسماء المتغيرات إنجليزي، والنصوص اللي المستخدم يشوفها عامية مصرية
- استخدم `el()` من [`js/lib.js`](js/lib.js) في بناء العناصر بدل `innerHTML` — إلا لو النص متأمّن بـ `esc()` أو `linkify()`

**قاعدة أمان مهمة:** أي نص جاي من مستخدم لازم يعدّي على `esc()` أو `linkify()` قبل ما يتحط كـ HTML. لو مش متأكد، استخدم `text:` بدل `html:` في `el()`.

**قواعد البيانات:** لو ضفت جدول أو غيّرت في السكيما:
1. طبّق الـ migration على مشروع Supabase بتاعك (مش بتاع المشروع)
2. حدّث [`supabase/schema.sql`](supabase/schema.sql) و [`docs/DATABASE.md`](docs/DATABASE.md)
3. اتأكد إن RLS متفعّل وإن `get_advisors` مفيهوش تحذيرات جديدة

### 4. افحص شغلك

```bash
npm run check              # كل الـ imports بتتحل + CSS سليم + الملفات الأساسية موجودة
node scripts/smoke.mjs     # بيفتح الصفحة بمتصفح حقيقي ويتأكد إنها اترندرت ووصلت لـ Supabase
```

**وبعدين جرّبها بإيدك في المتصفح.** الفحص الآلي بيثبت إن الكود بيشتغل، مش إنه بيعمل الصح.

قبل ما تبعت، اتأكد إنك جربت:
- ✅ في الوضع النهاري والليلي
- ✅ على شاشة موبايل (390px) وشاشة كبيرة
- ✅ إن الحاجة اللي عدّلتها شغالة فعلًا مش بس بتفتح

### 5. اعمل commit

بنستخدم [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: إضافة الردود على الستوريز
fix: عداد اللايكات كان بينقص تحت الصفر
style: تكبير الأزرار على الموبايل
docs: توضيح خطوات الرفع على Cloudflare
db: إضافة جدول story_replies مع RLS
perf: تقليل استعلامات الفيد من 4 لـ 2
```

### 6. افتح PR
املا القالب، وحط **صور قبل وبعد** لو التغيير في الشكل — دي أهم حاجة في مشروع زي ده.

## ✦ خريطة المشروع

عايز تعدّل حاجة؟ ابدأ من هنا:

| عايز تغير… | روح على |
| --- | --- |
| الألوان، الظلال، الخطوط، الأزرار | [`css/design.css`](css/design.css) |
| الهيدر، القوائم، الريسبونسڤ | [`css/layout.css`](css/layout.css) |
| شكل البوست، الستوري، الشات | [`css/components.css`](css/components.css) |
| شكل كارت البوست ومنطق اللايك | [`js/components.js`](js/components.js) |
| أي استعلام أو تعامل مع الداتابيز | [`js/sb.js`](js/sb.js) |
| صفحة كاملة (فيد، بروفايل، إكسبلور) | [`js/pages.js`](js/pages.js) |
| الراوتر، الشل، الودجتس الجانبية | [`js/app.js`](js/app.js) |
| الستوريز | [`js/stories.js`](js/stories.js) |
| الرسايل | [`js/messages.js`](js/messages.js) |

فيه شرح أعمق في [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md).
