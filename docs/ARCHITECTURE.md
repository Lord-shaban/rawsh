# البنية المعمارية 🏗️

إزاي روش متبني، وليه اتبنى كده.

---

## الفكرة الأساسية

روش **صفحة HTML واحدة** بتتكلم مع **Postgres** مباشرة. مفيش باك إند بينهم.

```
┌──────────────────────────────────────────────────────────┐
│                    المتصفح (الكلاينت)                     │
│                                                          │
│   index.html  →  js/app.js (راوتر بالهاش)                │
│         │                                                │
│         ├── pages.js ──┐                                 │
│         ├── stories.js ├──→ components.js (كارت البوست…)  │
│         ├── messages.js┤                                 │
│         └── compose.js ─┘                                │
│                    │                                     │
│                 js/sb.js  ← الطبقة الوحيدة اللي بتكلم برة │
└────────────────────┼─────────────────────────────────────┘
                     │  HTTPS + WebSocket
┌────────────────────┼─────────────────────────────────────┐
│                 Supabase                                 │
│                                                          │
│  PostgREST ──→ ┌─────────────────────────────────────┐   │
│  Realtime  ──→ │  Postgres                           │   │
│  Auth      ──→ │   • 16 جدول                         │   │
│  Storage   ──→ │   • Triggers (كل منطق العدادات)      │   │
│                │   • RPCs (الفيد، التريند، التصويت)   │   │
│                │   • RLS (كل الأمان)                  │   │
│                └─────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────┘
```

**النتيجة:** الرفع = نسخ ملفات على CDN. مفيش سيرفر يتصان، مفيش container، مفيش CI معقد.

---

## ليه من غير framework؟

مش عناد — قرار مبني على شكل المشروع:

| السبب | التفصيل |
| --- | --- |
| **صفر build** | تعدّل ملف وتعمل refresh. مفيش `npm run build` ولا انتظار |
| **الحجم** | التطبيق كله ~160KB كود + Supabase SDK من CDN |
| **العمر الافتراضي** | مفيش تبعيات تقدم أو تكسر. الكود ده هيشتغل بعد ٥ سنين زي النهارده |
| **الوضوح** | حد يفتح `js/pages.js` يفهم الفيد بيتبني إزاي من غير ما يعرف hooks ولا reactivity |

**الثمن:** مفيش reactive state تلقائي. بنعوّضه بـ event bus بسيط في `store` (شوف تحت).

## طبقات الكود

### `js/config.js`
رابط Supabase والمفتاح العام وحدود التطبيق (طول البوست، عدد الصور…). أول ملف تفتحه لو هتربط مشروع Supabase تاني.

### `js/lib.js` — الأدوات
مفيهاش أي علاقة بالمشروع نفسه. ممكن تنسخها في أي مشروع تاني:
- `el()` — بناء عناصر DOM بشكل مختصر وآمن
- `esc()` / `linkify()` — تأمين نصوص المستخدم + تحويل الروابط والهاشتاجات والمنشنات لعناصر قابلة للضغط
- `modal()` / `confirmDlg()` / `dropMenu()` / `lightbox()` — واجهات جاهزة
- `toast()` — الإشعارات المنبثقة
- `sfx` — أصوات ريترو مولّدة بـ Web Audio (مفيش ملفات صوت أصلًا)
- `compressImage()` — ضغط الصور في المتصفح قبل الرفع
- `timeAgo()` / `nfmt()` — تنسيق عربي للوقت والأرقام

### `js/sb.js` — طبقة البيانات
**الملف الوحيد اللي بيتكلم مع الخارج.** أي صفحة عايزة داتا بتنادي `api.*` — مفيش صفحة بتنادي `supabase` مباشرة.

بيحتوي على:

**`store`** — الحالة العامة + ناقل أحداث بسيط:
```js
store.me            // البروفايل بتاعي
store.followingIds  // Set بمين أنا متابعه
store.online        // Set بمين أونلاين دلوقتي
store.unreadNotifs  // عدد الإشعارات الجديدة

store.on('badges', fn)   // اشترك في حدث
store.emit('badges')     // أطلق حدث
```
الاشتراك بيرجّع دالة إلغاء، وكل صفحة بتجمّع دوال الإلغاء بتاعتها وترجّعها للراوتر عشان ينضّف لما تتقفل.

**`api`** — كل العمليات مجمّعة (`createPost`, `like`, `sendMessage`…). الأخطاء بترجع خام وبتتحول لعربي بـ `arErr()`.

**كاش البروفايلات** — `profilesByIds()` بتجيب اللي ناقص بس، وبتخزّن الباقي. ده بيمنع عشرات الاستعلامات المكررة في الفيد.

**تجميع البوستات** — `assemblePosts()` بتاخد صفوف خام وترجّعها كاملة: المؤلف، البوست المقتبس، وفلاجز التفاعل (`_liked`, `_reposted`, `_bookmarked`, `_myVote`) — كل ده في **٤ استعلامات ثابتة** مهما كان عدد البوستات، مش استعلام لكل بوست.

**طابور المشاهدات** — المشاهدات بتتجمّع في `Set` وتتبعت كل ٢.٥ ثانية دفعة واحدة بـ `register_views(uuid[])` بدل طلب لكل بوست.

### `js/components.js` — العناصر المشتركة
`postCard()` هو قلب التطبيق: بيبني كارت البوست كامل بالميديا والاستطلاع والاقتباس وأزرار التفاعل.

**التحديث المتفائل (Optimistic UI):** لما تدوس لايك، العداد بيزيد فورًا والصوت بيشتغل، وبعدين الطلب بيتبعت. لو فشل، بيترجع زي ما كان.

`observeViews()` بتستخدم `IntersectionObserver` عشان تسجّل المشاهدة لما البوست يظهر فعلًا على الشاشة بنسبة 45%.

`infiniteScroll()` بتحمّل الصفحة اللي بعدها لما تقرّب من آخر القايمة بـ 700px.

### `js/pages.js` وأخواتها — الصفحات
كل صفحة **دالة بتاخد عنصر وترجّع دالة تنظيف**:

```js
export function feedPage(root) {
  const cleanups = [];
  // … ابني الصفحة، سجّل مستمعين
  cleanups.push(store.on('newPost', handler));
  return () => cleanups.forEach((f) => f?.());
}
```

الراوتر بينادي دالة التنظيف قبل ما يفتح صفحة جديدة. ده اللي بيمنع تسريب المستمعين والمؤقتات.

### `js/app.js` — الراوتر والشل
راوتر بالهاش (`#/u/ahmed`) مبني على مصفوفة regex. بيستخدم `document.startViewTransition` لو المتصفح بيدعمها.

الهاش مش مجرد تفضيل — هو اللي بيخلي التطبيق يشتغل على أي استضافة static من غير أي إعدادات rewrite.

## الحدود: إيه في الكلاينت وإيه في الداتابيز

القاعدة: **أي حاجة لو المستخدم زوّرها هتفرق — لازم تكون في الداتابيز.**

| الحاجة | فين | ليه |
| --- | --- | --- |
| ترتيب الفيد | 🗄️ الداتابيز (`feed_for_you`) | الترتيب لازم يكون واحد للكل ومايتلعبش فيه |
| عدادات اللايكات والتعليقات | 🗄️ Triggers | لو الكلاينت هو اللي يزوّد، أي حد يقدر يكتب أي رقم |
| استخراج الهاشتاجات والمنشنات | 🗄️ Trigger | عشان يشتغل مهما كان مصدر البوست |
| الإشعارات | 🗄️ Triggers | مايتنسيش، ومايتزيّفش |
| منع التصويت المرتين | 🗄️ مفتاح أساسي + دالة | الكلاينت مش مصدر ثقة |
| تنسيق الوقت والأرقام بالعربي | 💻 الكلاينت | شكل بحت |
| التحديث المتفائل للايك | 💻 الكلاينت | إحساس سريع — والداتابيز هي الحقيقة النهائية |
| ضغط الصور | 💻 الكلاينت | يوفّر باندويث ومساحة قبل الرفع أصلًا |

## اللحظي (Realtime)

٣ قنوات شغالة مع بعض:

| القناة | الآلية | بتعمل إيه |
| --- | --- | --- |
| `user-<id>` | Postgres Changes | إشعارات جديدة + تحديثات المحادثات (توست + صوت + بادج) |
| `feed-posts` | Postgres Changes | أي بوست جديد → بيل "✦ ٣ بوستات جديدة" فوق الفيد |
| `online` | Presence | مين متواجد دلوقتي (النقطة الخضرا + ودجت الأونلاين) |
| `conv-<id>` | Changes + Broadcast | رسايل المحادثة المفتوحة + مؤشر "بيكتب دلوقتي" |

مؤشر الكتابة بيستخدم **Broadcast** مش جدول — لأنه بيانات مؤقتة ملهاش لازمة تتخزن.

## قرارات ملهاش رجعة (وليه)

**الرسايل: جدول محادثات بدل بحث في الرسايل**
`conversations` فيه `unique(user1, user2)` مع `check (user1 < user2)`. الترتيب ده بيضمن إن محادثة أحمد ومحمد ليها صف واحد بس مهما كان مين اللي بدأ.

**التعليقات: مستوى واحد من الردود**
تعليق → ردود عليه، وبس. التداخل اللانهائي بيعقّد الاستعلام والواجهة من غير فايدة حقيقية في منصة زي دي.

**الاستطلاعات: العدادات في JSONB جوه البوست**
بدل جدول منفصل يتعدّ كل مرة. جدول `poll_votes` موجود عشان يمنع التصويت المزدوج بس، والعدادات محفوظة في `posts.poll` وبتتحدث ذريًا في دالة `poll_vote` مع `FOR UPDATE`.

**البوستات: `media` كـ JSONB مش جدول**
البوست بيتعرض كوحدة واحدة دايمًا، ومفيش استعلام بيدور على صورة لوحدها. JSONB بيوفّر join في كل عرض.

## الأداء

- **٤ استعلامات لأي فيد** مهما كان عدد البوستات (بفضل التجميع الدفعي)
- **كاش البروفايلات** يمنع تكرار جلب نفس المستخدم
- **المشاهدات بالدفعات** كل ٢.٥ ثانية بدل طلب لكل بوست
- **الصور بتتضغط في المتصفح** لعرض 1600px وجودة 85% قبل الرفع
- **الفيديوهات بتتوقف تلقائيًا** لما تخرج من الشاشة (`IntersectionObserver`)
- **فهارس** على كل عمود بيتفلتر أو يترتب بيه، و`pg_trgm` للبحث النصي
